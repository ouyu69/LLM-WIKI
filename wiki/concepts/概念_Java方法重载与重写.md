# 概念：Java 方法重载与重写

## 重载
重载发生在同一个类中，多个方法名称相同但参数列表不同。参数列表不同可以是参数个数、类型或顺序不同；仅返回值不同不能构成重载。

## 重写
重写发生在继承关系中，子类对父类可继承方法提供新的实现。重写要求方法名、参数列表保持一致，返回类型通常相同或为协变返回类型。

## 关键区别
- 绑定时机：重载主要在编译期根据参数静态决定；重写体现运行时动态分派。
- 使用目的：重载提升 API 易用性；重写实现多态和行为扩展。
- 访问权限：重写时子类方法访问权限不能比父类更严格。
- 异常约束：重写方法不能抛出比父类方法更宽泛的受检查异常。
