# 概念：Java 泛型

## 定义
泛型允许类、接口或方法在定义时使用类型参数，在使用时再指定具体类型，从而提高类型安全和代码复用性。

## 使用方式
- 泛型类：例如 `class Box<T>`。
- 泛型接口：例如 `interface Repository<T>`。
- 泛型方法：方法声明自己的类型参数，例如 `<T> T copy(T value)`。

## 核心理解
- 泛型让编译器在编译期检查类型，减少显式强制类型转换。
- Java 泛型主要通过类型擦除实现，运行期很多泛型参数信息会被擦除。
- 泛型不能直接使用基本类型，需要使用包装类型。

## 常见场景
- 集合类型约束，如 `List<String>`。
- 通用返回结果、分页对象、工具类。
- 框架中的类型约束与扩展点设计。
