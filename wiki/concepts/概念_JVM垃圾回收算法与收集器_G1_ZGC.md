# 概念：JVM 垃圾回收算法与收集器（G1 / ZGC）

## 常见 GC 算法
- 标记-清除：实现简单，但可能产生内存碎片。
- 复制：以空间换时间，适合对象存活率低的区域。
- 标记-整理：减少碎片，适合存活对象较多的区域。
- 分代收集：结合对象生命周期特点，将不同算法应用到不同代。

## 默认收集器（面试关注点）
- JDK 8：常见默认是 Parallel GC（不同发行版/参数下可能不同，排查以实际启动参数为准）。
- JDK 9+：常见默认是 G1 GC。

## G1（Garbage-First）
- 核心思路：堆划分为多个 Region，后台维护“回收价值优先列表”，在停顿预算内优先回收收益最大的 Region。
- 目标：在可控停顿时间下取得较好的整体吞吐与可预测性，适合较大堆场景。

## ZGC
- 目标：将暂停时间控制在更低水平，并尽量不随堆大小线性增长（但可能牺牲吞吐）。
- 演进：Java 11 引入，后续版本逐步完善；在 Java 21 引入分代 ZGC 以进一步降低暂停时间。
