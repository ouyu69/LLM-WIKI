## String 、 StringBuffer、Stringbuilder的区别

|               | 是否可变 |        是否线程安全         | 性能  |
| :-----------: | :--: | :-------------------: | :-: |
|    String     | 不可变  |           是           |  低  |
| StringBuffer  |  可变  | 是（使用了 `synchronized`） | 较高  |
| StringBuilder |  可变  |           否           |  高  |

## 自动装箱和自动拆箱

Java 中的 自动拆装箱 是一个语法糖，它能让 基本数据类型 与 引用类型 无缝转换，但也引入了性能问题。装箱会创建对象，频繁创建对象会带来内存压力，同时对 null 进行拆箱还会直接抛出 空指针异常。

## 面向对象三大特征

- **封装**：隐藏对象的属性和实现细节，仅对外公开接口
- **继承**：继承就是子类继承父类的行为和特征
- **多态**：同一个行为具有不同的表现形态

---
## Java 强制类型转换原理

- **向上转型**：将一个 子类引用 转换为 父类引用，这种转换是隐式的
- **向下转型**：将一个 父类引用 转换为 子类引用，这种转换是显示的，主要用于调用子类特有的方法，在方法不被重载的基础上

## 抽象类和接口的对比

### 共同点

- 都不能实例化
- 都可以包含抽象方法（接口在Java 8+可包含默认方法）

### 不同点

| 特性         | 抽象类                     | 接口                     |
|--------------|---------------------------|-------------------------|
| 定义         | `abstract class`          | `interface`             |
| 继承方式     | 单继承                    | 多实现                  |
| 构造器       | 可以定义构造器            | 无构造器                |
| 字段         | 可包含任意类型字段        | 字段默认为 `public static final` |
| 方法         | 可有抽象/具体方法         | Java 8+可含默认/静态方法 |

## 什么是值传递，什么是引用传递

- **值传递**：函数调用时，实际参数的值被复制到函数的形式参数当中，函数内部对形式参数的修改不会影响实际参数。
- **引用传递**：指在调用函数时，直接将参数的地址传到函数当中，函数中对参数进行修改，会对实际参数产生影响
